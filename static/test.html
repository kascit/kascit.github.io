This page has no CSS links and no markup for the navbar/sidebar. The script auto-injected the CSS, fetched the layout shell, wrapped this content in the drawer, and configured everything from window.SiteNavConfig.

Minimal usage

<script src="https://dhanur.me/js/shell.js" defer></script>

That's literally it. The script injects CSS, fetches the navbar shell, wraps your page, and handles theming.

Full config example 

<script>
  window.SiteNavConfig = {
    // Layout: "full" (drawer + sidebar) or "navbar" (top bar only)
    mode: "full",

    // Navigation items
    nav: [
      { name: "Home", url: "/", icon: "fa-solid fa-house" },
      { name: "Docs", url: "/docs/", icon: "fa-solid fa-book" }
    ],
    sidebarNav: [ ... ],   // optional, falls back to nav
    activePath: "/docs/",  // highlight active sidebar item

    // Branding overrides
    logo: { href: "/", html: "<img src='/logo.png' class='h-8'>" },
    favicon: "/icon.png",
    badge: { text: "BETA", class: "badge-warning" },

    // Chrome visibility
    showSearch: false,      // default: false (subdomains can't search)
    showAppsGrid: true,     // default: true
    showThemeToggle: true   // default: true
  };
</script>

Config reference

Option Type Default Description
mode "full" | "navbar" "full" Full drawer layout or navbar-only
nav Array Main site nav Header navigation items
sidebarNav Array Falls back to nav Sidebar navigation with optional children
activePath string location.pathname Path to highlight as active
logo { href, html } Main site logo Override logo link and content
favicon string Main site favicon Path to custom favicon
badge { text, class } None Badge next to logo (e.g. "BETA")
showSearch boolean false Show search bar (needs main site index)
showAppsGrid boolean true Show the apps grid dropdown
showThemeToggle boolean true Show the theme toggle button
